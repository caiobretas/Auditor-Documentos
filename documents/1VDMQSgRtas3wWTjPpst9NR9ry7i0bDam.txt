API de Carteira: além do login nativo Web3 com carteiras 
de auto-custódia, os usuários da Lodyge também terão a 
opção de criar uma carteira semi-custodiada utilizando o 
Gmail ou outras formas de login baseadas na Web2.
Os usuários terão a opção de conectar a carteira semi-
custodiada usando formas de autenticação Web2. Em 
ambientes externos, como o OpenSea, eles podem utilizar 
o Wallet Connect para estabelecer a conexão.
Se o usuário desejar, têm a opção de migrar para carteiras 
de auto-custódia, como a Metamask, ao transferir seus 
ativos.
API de Pagamento: permita que usuários utilizem meios 
de pagamento como PIX e Cartão de Crédito para adquirir 
ativos digitais.
API de Marketplace: tenha um mercado secundário de 
NFTs dentro das suas plataformas, tirando a necessidade 
de expor seus usuários ao OpenSea ou outros mercados 
externos.
API de Smart Contract: crie suas coleções em um mesmo 
smart contract de Polygon (ou qualquer rede EVM) e 
ofereça ações “token-gated” por meio da propriedade e 
metadado dessas coleções. 
API de Credentials: gere NFTs baseados em jornadas 
gamificadas, envolvendo o cumprimento de requisitos 
ativos (executar uma ação específica, como seguir um 
perfil no Twitter), ou passivos (possuir um NFT da Reserva).
